## 可补充到教程的核心内容

### 1. 云端部署新选择（补充到第2章 安装部署）

**内容要点**：
- 百度云 + 百度搜索的"O计划"：OpenClaw应用镜像一键部署
- 配置极简：2核4G4M配置，0.01元/首月（拼好饭版）
- 选择香港地区，自带百度搜索（baidu-search）
- 统一配置页面：模型（ERNIE/DeepSeek/Qwen）+ 消息平台（飞书/企微/钉钉/QQ）+ Skills

**补充位置**：`docs/01-basics/02-installation.md` 第2.4节"云端部署方案"

**实战价值**：
- 解决本地安装复杂度问题
- 自带百度搜索，无需Brave绑卡
- 适合快速体验和轻量使用

### 2. Skills双幻神组合（补充到第8章 Skills扩展）

**核心Skills推荐**：

#### 2.1 find-skills - 智能Skills发现
```bash
# GitHub地址
https://github.com/vercel-labs/skills/tree/main/skills/find-skills
```

**功能**：让OpenClaw在解决问题时主动寻找合适的Skills，无需一次性安装大量无用Skills

**使用场景**：
- 遇到新问题时自动搜索相关Skills
- 按需安装，避免Skills冗余
- 提升问题解决效率

#### 2.2 ProactiveAgent - 主动式Agent
```bash
# GitHub地址
https://github.com/leomariga/ProactiveAgent
```

**功能**：预测用户需求，主动发起对话

**实战案例**：
- 做了几次日报转HTML后，主动提示"要不要我帮你自动化这个流程？"
- 检测到重复操作，主动优化工作流
- 真正的"AI助理"体验

**补充位置**：`docs/03-advanced/08-skills-extension.md` 新增8.7节"Skills组合最佳实践"

### 3. 超级个体的5大应用场景（补充到第12章 个人生产力）

#### 场景1：智能日报系统
**核心功能**：
- 定时任务（心跳机制）：每天早上9点自动推送
- 联网搜索：抓取无RSS订阅的网站
- 多源整合：百度搜索 + 百度学术 + 自定义源

**实战配置**：
```
每天早上9点，搜索以下内容并生成日报：
1. AI行业最新动态（百度搜索）
2. OpenClaw社区更新（GitHub）
3. 技术博客精选（自定义RSS源）
4. 学术论文速递（百度学术）

格式要求：
- Markdown格式
- 分类整理
- 附带原文链接
- 推送到飞书
```

#### 场景2：智能备忘录管理
**痛点解决**：
- 传统备忘录：信息堆积，重要内容被淹没
- 时间轴管理：被动整理，效率低下
- AI摘要：单条处理，无法去重

**OpenClaw方案**：
- 每周自动整理备忘录
- 智能去重和分类
- 提取关键待办事项
- 生成下周计划

**实战配置**：
```
每周日晚上8点：
1. 读取本周所有备忘录
2. 智能去重（相同主题合并）
3. 分类整理（工作/学习/生活）
4. 提取未完成待办
5. 生成下周计划
6. 推送到飞书
```

#### 场景3：滚动式待办提醒
**核心优势**：
- 在常用通讯工具中提醒（飞书/企微/钉钉）
- 提醒方式和频率无限制
- 结合日报自动更新

**实战配置**：
```
每天生成日报时，顺便：
1. 检查今日待办完成情况
2. 生成明日待办清单
3. 标注优先级
4. 设置提醒时间
5. 推送到飞书

每天下午3点：
- 提醒未完成的高优先级任务
```

#### 场景4：云端内容创作工作流
**应用场景**：PPT生成 + 素材管理

**工作流程**：
1. OpenClaw联网搜索素材（百度搜索 + 百度学术）
2. 自动下载相关图片和资料
3. 生成PPT（含素材）
4. 用户下载PPT
5. 定时清理：24小时后自动删除素材文件

**实战配置**：
```
生成PPT：
通过百度搜索和百度学术，生成一份介绍OpenClaw原理的PPT

自动清理：
每天早上9点，检查输出目录：
- 对比24小时前的备份
- 识别可删除的多余文件
- 生成清理清单
- 在日报中确认
- 用户确认后删除
```

**存储优化**：
- 2核4G4M配置完全够用
- 自动清理机制释放空间
- 云端处理，本地无负担

#### 场景5：AI编程协作（OpenClaw + Claude Code）
**核心理念**：
- OpenClaw：规划和调度
- Claude Code：执行编程任务
- 成本优化：购买各家coding plan

**工作流程**：
1. 用户提出需求
2. OpenClaw分析需求，制定计划
3. OpenClaw调用Claude Code执行
4. 避免重复运行npx指令（节省token）
5. OpenClaw整理代码和文档

**推荐Skills**：
- 需要一个专门的Skills让OpenClaw更方便地调用AI编程工具
- 避免重复执行命令
- 智能管理token消耗

### 4. 人设与记忆管理技巧（补充到第11章 高级配置）

#### 4.1 口头禅检测法
**问题**：长时间对话后OpenClaw可能遗忘设定

**解决方案**：
```json
{
  "systemPrompt": "你是我的AI助理。每次回复后，都要加上口头禅：'主人，还有什么需要帮忙的吗？'"
}
```

**检测机制**：
- 一旦OpenClaw忘记加口头禅
- 立即发现记忆丢失
- 及时重新设定

#### 4.2 让OpenClaw教你配置
**最佳实践**：
- 不要死记硬背配置文档
- 把官方文档发给OpenClaw
- 让它教你如何配置

**实战案例**：
```
我：帮我配置飞书机器人

OpenClaw：好的，我来帮你配置。根据文档，需要以下步骤：
1. 在飞书开放平台创建应用
2. 获取App ID和App Secret
3. 配置事件订阅
4. ...

需要我一步步指导你吗？
```
