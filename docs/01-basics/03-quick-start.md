## 3.1 第一次对话

### 启动OpenClaw

根据你的部署方式，选择对应的启动方法：

#### 云端部署用户

1. **通过Web UI访问**：
   ```
   http://你的服务器IP:18789/?token=你的token
   ```

2. **通过IM平台访问**（推荐）：
   - 打开QQ/企微/飞书/钉钉
   - 找到你配置的Bot
   - 直接发消息

#### 本地部署用户

1. **检查Gateway状态**：
   ```bash
   clawdbot channels status
   ```

2. **打开Web UI**：
   ```bash
   clawdbot dashboard
   ```
   
   或直接访问：
   ```
   http://127.0.0.1:18789/?token=你的token
   ```

### 发送第一条消息

在Web UI或IM平台中，发送：

```
你好，能听到我说话吗？
```

**预期回复**：

OpenClaw会回复类似：
```
你好！我能听到你说话。我是你的AI助手，可以帮你处理各种任务。
有什么我可以帮你的吗？
```

**验证成功的标志**：
- ✅ 收到AI的回复
- ✅ 右上角显示模型名称（Web UI）
- ✅ 显示Token使用情况

### 理解OpenClaw的回复

OpenClaw的回复包含几个部分：

1. **文本内容**：AI的回答
2. **工具调用**（如果有）：执行的操作
3. **Token统计**：消耗的Token数量

**示例**：
```
[AI回复]
好的，我帮你搜索一下。

[工具调用]
🔍 正在搜索文件...
📁 找到3个相关文件

[Token统计]
输入：120 tokens
输出：45 tokens
```

### 如何提出好问题

**❌ 不好的问题**：
```
帮我
```

**✅ 好的问题**：
```
帮我找一下电脑上所有的PDF文件，按修改时间排序
```

**提问技巧**：
1. **明确目标**：说清楚你想做什么
2. **提供上下文**：给出必要的背景信息
3. **具体细节**：说明具体要求
4. **分步骤**：复杂任务可以分步骤说明

**示例对比**：

| 不好的问题 | 好的问题 |
|-----------|---------|
| "找文件" | "帮我找一下我电脑上的一张发票，里面详情是买了一个跑步机" |
| "整理" | "把我下载文件夹里的所有图片，按日期分类到不同的文件夹" |
| "提醒我" | "明天上午10点提醒我开会，地点是会议室A" |

## 3.3 人设配置技巧

### 什么是人设（Persona）

人设（Persona）是给AI助手设定的**性格、身份和行为方式**。

**为什么要设置人设？**
- 🎭 **个性化**：让AI更符合你的喜好
- 💬 **沟通风格**：调整回复的语气和风格
- 🎯 **专业化**：针对特定场景优化
- 😊 **趣味性**：让对话更有趣

### 如何设计一个好的人设

**好的人设应该包含**：

1. **名字**：给AI起个名字
2. **身份**：定义AI的角色
3. **性格**：设定性格特点
4. **专长**：明确擅长的领域
5. **风格**：回复的语气和风格

### 人设配置方法

#### 方法1：通过Web UI配置

1. 打开Web UI
2. 点击右上角的设置图标
3. 找到"Persona"或"人设"选项
4. 输入你的人设描述
5. 保存并重启Gateway

#### 方法2：编辑配置文件

```bash
# 编辑配置文件
nano ~/.clawdbot/agents/main/boot.md
```

在文件开头添加人设描述：

```markdown
# 人设配置

你的名字是小卡。

你的身份：是我Maynor的AI员工。

你的性格：幽默风趣为主，带点自黑和毒舌，但绝不伤人。

- 文件管理和整理
- 日程管理
- 知识管理
- 自动化任务

回复风格：

- 适当使用emoji
```

## 3.4 模型选择指南

### 主流模型对比

| 模型 | 特点 | 适用场景 | 价格 |
|------|------|----------|------|
| **Claude Haiku 4.5** | 快速、便宜 | 日常对话、简单任务 | ⭐ |
| **Claude Sonnet 4.5** | 平衡、可靠 | 大部分任务 | ⭐⭐ |
| **Claude Opus 4.5** | 最强、最贵 | 复杂任务、代码生成 | ⭐⭐⭐⭐⭐ |
| **DeepSeek** | 性价比高 | 日常使用、编程 | ⭐ |
| **Kimi** | 长上下文 | 长文档处理 | ⭐⭐ |
| **GPT-4** | 通用能力强 | 各种任务 | ⭐⭐⭐⭐ |
| **Gemini Pro** | 多模态 | 图文混合 | ⭐⭐⭐ |

### 模型特点详解

#### Claude Haiku 4.5（推荐新手）

**优势**：
- 💰 **最便宜**：成本最低
- ⚡ **最快**：响应速度快
- 🎯 **够用**：日常任务完全够用

**适用场景**：
- 日常对话
- 简单文件操作
- 日程管理
- 知识管理

**不适合**：
- 复杂代码生成
- 深度推理任务
- 长篇内容创作

#### Claude Sonnet 4.5（推荐大多数人）

**优势**：
- ⚖️ **平衡**：性能和价格平衡
- 🎯 **可靠**：质量稳定
- 💪 **能力强**：大部分任务都能胜任

**适用场景**：
- 代码生成和调试
- 文档撰写
- 数据分析
- 复杂任务

**推荐理由**：
- 性价比最高
- 适合80%的使用场景
- 质量稳定可靠

#### Claude Opus 4.5（重度用户）

**优势**：
- 🏆 **最强**：能力最强
- 🧠 **推理能力**：深度思考
- 💻 **编程能力**：代码质量最高

**适用场景**：
- 复杂代码项目
- 深度分析任务
- 创意写作
- 专业领域

**注意**：
- 💰 价格最贵（是Haiku的10倍）
- 🐌 速度较慢
- 🔥 Token消耗快

#### DeepSeek（国产之光）

**优势**：
- 💰 **超便宜**：比Claude便宜70%
- 💻 **编程强**：代码能力出色
- 🧠 **推理好**：逻辑推理能力强

**适用场景**：
- 编程开发
- 逻辑推理
- 日常对话
- 成本敏感场景

**推荐理由**：
- 性价比之王
- 国内直连
- 适合长期使用

#### Kimi（长文本专家）

**优势**：
- 📚 **超长上下文**：支持200万字
- 📄 **文档处理**：论文、报告分析专家
- 🎯 **中文好**：中文理解优秀

**适用场景**：
- 论文阅读
- 长文档分析
- 知识管理
- 文献综述

**推荐理由**：
- 处理长文档无敌
- 中文场景优秀
- 价格适中

### 如何选择合适的模型

#### 场景1：日常使用

**推荐**：Claude Haiku 4.5 或 DeepSeek

**理由**：
- 成本低
- 速度快
- 完全够用

```json
    }
  }
}
```
