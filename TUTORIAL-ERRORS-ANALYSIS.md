## ❌ 严重错误（需立即修复）

### 1. 版本号格式不规范 🔢

**错误描述**: 版本号使用日期格式（2026.2.9），不符合语义化版本规范。

**出现位置**:
- "2026.2.9" (推荐版本)
- "2026.2.12" (不推荐版本)
- "2026.2.1-zh.3" (中文版)

**问题影响**:
- ❌ 不符合 SemVer 规范（应该是 MAJOR.MINOR.PATCH）
- ❌ 用户难以理解版本之间的关系
- ❌ 与 npm 版本管理不兼容

**正确的版本号格式**:
```
语义化版本: MAJOR.MINOR.PATCH
示例: 1.2.9 或 v1.2.9

当前使用（不规范）:
- 2026.2.9 (看起来像日期，不像版本号)
- 2026.2.1-zh.3 (语言标识不应该在版本号中)
```

**修复建议**:
1. 如果 2026.2.9 是官方版本号，需要在文档中明确说明这是特殊的版本号格式
2. 或者建议官方改用标准的语义化版本号
3. 在教程中添加版本号格式说明

---

### 2. 项目名称演变说明不清晰 📛

**错误描述**: 教程中同时提到多个版本号，且版本号格式不统一。

**出现位置**:
- "2026.2.9" (推荐版本)
- "2026.2.12" (不推荐版本)
- "2026.2.1-zh.3" (中文版)
- "openclaw@latest"

**问题影响**:
- ❌ 用户不知道该安装哪个版本
- ❌ 版本号格式不符合语义化版本规范
- ❌ 中文版和国际版版本号不一致

**正确的版本号格式**:
```
语义化版本: MAJOR.MINOR.PATCH
示例: 1.2.9 或 v1.2.9

不应该使用:
- 2026.2.9 (年份不应该作为主版本号)
- 2026.2.1-zh.3 (语言不应该在版本号中)
```

**修复建议**:
1. 统一使用 `v1.x.x` 格式
2. 明确标注当前稳定版本
3. 删除"不推荐版本"的说法（应该说"已知问题版本"）

---

### 3. 项目名称混乱 📛

**错误描述**: 教程中同时使用多个项目名称。

**出现位置**:
- OpenClaw (当前名称)
- Clawdbot (旧名称)
- Moltbot (中间名称)
- ClawHub (技能市场)

**问题影响**:
- ❌ 新手不知道这些是不是同一个项目
- ❌ 搜索时找不到正确的资源
- ❌ 命令行工具名称不一致

**修复建议**:
1. 在教程开头明确说明项目名称演变历史
2. 统一使用 OpenClaw 作为项目名称
3. 命令行工具统一使用 `openclaw` 命令

## ⚠️ 中等错误（影响使用体验）

### 4. 安装命令不一致 💻

**错误描述**: 不同章节给出的安装命令不一致。

**出现位置**:
```bash
# 第2章
curl -fsSL https://openclaw.ai/install.sh | bash

# 国内版
curl -fsSL https://clawd.org.cn/install.sh | bash

# npm安装
npm install -g openclaw@latest
npm install -g openclaw-cn@latest
npm install -g @qingchencloud/openclaw-zh@latest
```

**问题影响**:
- ❌ 用户不知道该用哪个命令
- ❌ 不同命令安装的可能是不同版本
- ❌ 国内外用户混淆

**修复建议**:
1. 明确区分国际版和国内版
2. 提供清晰的安装决策树
3. 说明不同安装方式的区别

---

**问题影响**:
- ❌ 用户找不到配置文件
- ❌ 修改配置后不生效
- ❌ 多 Agent 配置混乱

**修复建议**:
1. 明确说明配置文件的层级结构
2. 提供配置文件位置的完整说明
3. 添加配置文件查找命令

### 7. API Key 配置混乱 🔑

**错误描述**: API Key 配置方式在不同章节说法不一致。

**出现位置**:
```bash

# 有时说用配置文件

# 有时说用命令
openclaw models auth add

```

**问题影响**:
- ❌ 用户不知道该用哪种方式
- ❌ 配置后不生效
- ❌ 多个 API Key 管理混乱

**修复建议**:
1. 明确推荐的配置方式
2. 说明不同方式的优先级
3. 提供完整的配置示例

**出现位置**:

2. 使用相对路径引用图片
